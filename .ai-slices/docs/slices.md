# Slice Manifest Template

Reference shape for the real `docs/slices.md` generated by `/plan`.

```markdown
# Slice Manifest

| Slice ID | Journey | Priority | Depends On | Modules | Pages | Schema Impact | Acceptance | Status |
|----------|---------|----------|------------|---------|-------|---------------|------------|--------|
| SLICE-001 | Visitor signs in and reaches the dashboard | Launch | - | AUTH, DASHBOARD | LoginPage, DashboardPage | No | Sign-in succeeds and dashboard data is visible | Planned |
```

Rules:

- `Slice ID` uses `SLICE-001` format.
- `Depends On` lists prior slice IDs or `-`.
- `Schema Impact` is `Yes` or `No`.
- `Status` is plan-level status, not execution-stage state.
